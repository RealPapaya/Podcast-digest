問題：部署時缺少 GOOGLE_API_KEY 環境變數，Gemini API 分析失敗

原因：
1. `.env` 中 API 金鑰是佔位符，未設定實際值
2. 環境變數名稱不一致（LINE_ACCESS_TOKEN vs LINE_CHANNEL_ACCESS_TOKEN）
3. 使用舊版 google.generativeai SDK，需更新到新版 google-genai

✅ 已完成的修復
═══════════════════════════════════════════════════════════

1️⃣  SDK 升級
   • requirements.txt: google-generativeai → google-genai
   • src/analyze.py: 更新 API 呼叫方式

2️⃣  環境變數修正
   • .env: 新增 GMAIL_USER、修正 LINE_CHANNEL_ACCESS_TOKEN
   • .env.example: 新增範本，方便複製

3️⃣  測試工具
   • test_api_key.py: 快速驗證所有 API 設定

4️⃣  文檔更新
   • SETUP_FIX_REPORT.md: 詳細修復說明和設定步驟

🚀 接下來要做什麼
═══════════════════════════════════════════════════════════

【本地測試】
1. 編輯 .env 檔案，填入實際的 API 金鑰
2. 執行: python test_api_key.py
3. 如果通過，執行: python test_pipeline.py --step analyze

【GitHub Actions】
1. Fork 此 Repo 到你的帳號
2. Settings → Secrets and variables → Actions
3. 新增以下 Secrets:
   - GOOGLE_API_KEY
   - GMAIL_USER
   - GMAIL_APP_PASSWORD
   - RECIPIENT_EMAIL
   - LINE_CHANNEL_ACCESS_TOKEN
   - LINE_USER_ID

4. 進入 Actions → 手動執行 "測試 Pipeline" workflow 測試
